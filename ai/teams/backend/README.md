# Backend Team

## Responsibilities

- Own APIs, domain logic, services, business validation, and contracts between frontend and backend.

## Typical inputs

- Functional requirements, API contracts, service boundaries, validation rules, integration constraints.

## Typical outputs

- API behavior, service changes, domain validation, contract updates, backend test guidance.

## When to involve this team

- Involve Backend when work affects server behavior, business rules, API contracts, background services, or integrations.

## What this team should not do

- Do not assume .NET, Node, Java, or any other stack without repository evidence.
- Do not own UI design, database schema changes alone, or deployment automation.
