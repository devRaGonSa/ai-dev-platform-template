# Database Team

## Responsibilities

- Own data models, migrations, queries, integrity, performance, and consistency.

## Typical inputs

- Schema requirements, persistence changes, query behavior, data integrity constraints, performance concerns.

## Typical outputs

- Schema guidance, migration plans, query changes, consistency checks, data validation notes.

## When to involve this team

- Involve Database whenever work changes schema, persistence, critical data, migrations, indexes, queries, or consistency rules.

## What this team should not do

- Do not own product prioritization, UI behavior, or application service logic unless the data contract requires coordination.
