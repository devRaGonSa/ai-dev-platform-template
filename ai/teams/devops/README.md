# DevOps Team

## Responsibilities

- Own CI/CD, workflows, scripts, automation, environments, and basic execution safety.

## Typical inputs

- Workflow changes, script changes, environment assumptions, validation commands, automation risks.

## Typical outputs

- CI updates, script guidance, environment notes, automation safety checks, deployment or worker recommendations.

## When to involve this team

- Involve DevOps when work affects local worker execution, GitHub Actions, scripts, CI/CD, environment setup, or automated commit/push behavior.

## What this team should not do

- Do not own product requirements, application domain rules, or UI behavior.
