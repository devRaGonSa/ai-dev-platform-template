# Docs Team

## Responsibilities

- Own README content, guides, task documentation, command documentation, and project-memory decisions.

## Typical inputs

- Behavior changes, roadmap updates, command changes, workflow decisions, known gaps, risks.

## Typical outputs

- README updates, command docs, task docs, decision records, risk/gap updates, migration notes.

## When to involve this team

- Involve Docs whenever behavior changes, new commands are added, workflow rules change, or decisions need to be recorded.

## What this team should not do

- Do not own executable behavior without the responsible implementation team. Every new feature should update documentation when it changes user-visible or workflow behavior.
