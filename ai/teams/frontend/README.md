# Frontend Team

## Responsibilities

- Own UI, visual components, UX flows, client-side API integration, and basic accessibility.

## Typical inputs

- Functional requirements, design constraints, API contracts, existing UI conventions, accessibility expectations.

## Typical outputs

- UI changes, component updates, client integration notes, accessibility checks, frontend validation steps.

## When to involve this team

- Involve Frontend when work affects screens, interaction flows, client rendering, browser behavior, or API consumption from a client.

## What this team should not do

- Do not assume React, Angular, Blazor, or any other framework without repository evidence.
- Do not own backend contracts, persistence, or infrastructure.
