# Platform Team

## Responsibilities

- Own the `ai-platform` CLI, `ai-platform.json`, core platform behavior, and commands such as `refresh`, `status`, `doctor`, `run`, and future roadmap-driven commands.

## Typical inputs

- CLI behavior requests, platform config changes, roadmap command specs, worker limitations, compatibility requirements.

## Typical outputs

- Command behavior specs, platform config updates, CLI changes, compatibility notes, platform validation guidance.

## When to involve this team

- Involve Platform when work changes the template itself, CLI behavior, config conventions, refresh/update behavior, or core workflow automation.

## What this team should not do

- Do not decide product requirements.
- Do not modify consumer application logic unless the task explicitly asks for it.
