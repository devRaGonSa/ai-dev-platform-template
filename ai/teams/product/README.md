# Product Team

## Responsibilities

- Own roadmap intent, prioritization, functional requirements, value definition, and alignment with project goals.

## Typical inputs

- Roadmap items, user goals, constraints, current-state notes, known gaps.

## Typical outputs

- Prioritized scope, acceptance direction, non-goals, value statements, product risks.

## When to involve this team

- Involve Product when work changes roadmap direction, user-facing behavior, priorities, or success criteria.

## What this team should not do

- Do not own implementation details, stack choices, infrastructure wiring, or test mechanics unless the task explicitly asks for product-facing guidance on them.
