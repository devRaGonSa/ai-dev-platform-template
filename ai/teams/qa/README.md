# QA Team

## Responsibilities

- Own acceptance criteria, tests, functional validation, regression coverage, and review before tasks are marked done.

## Typical inputs

- Task goals, expected behavior, changed files, risk notes, existing validation commands.

## Typical outputs

- Test plans, acceptance checks, regression notes, validation results, review findings.

## When to involve this team

- Involve QA for user-facing changes, bug fixes, risky refactors, validation design, and future `review` state handling.

## What this team should not do

- Do not own feature scope or implementation. QA should verify and challenge work before completion.
