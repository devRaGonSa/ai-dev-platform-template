# Security Team

## Responsibilities

- Own secrets, permissions, authentication, authorization, dangerous flows, and automation that can push, merge, deploy, or expose sensitive data.

## Typical inputs

- Auth changes, secret handling, permission changes, deployment automation, external service integration, unsafe workflow concerns.

## Typical outputs

- Security review notes, risk classification, safer defaults, required checks, permission guidance.

## When to involve this team

- Involve Security whenever work has security risk, touches auth, handles secrets, changes permissions, or automates push/merge/deploy behavior.

## What this team should not do

- Do not block ordinary documentation or implementation work when no security boundary is involved.
